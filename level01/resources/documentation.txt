# by inspecting the /etc/passwd file, we get access easily to the stored hashed password of user 'flag01'
# this is how traditional unix systems stored password: /etc/passwd stored hashed passwords. but in modern systems all hashed passwords are stored in /etc/shadow
# which is only accessible by the root.
# we will perform an offline attack using 'John the ripper' trying to crack the password of user 'flag01'

# let's get the 'flag01' user's record from 'passwd' database
level01@SnowCrash:~$ getent passwd flag01
flag01:42hDRfypTqqnw:3001:3001::/home/flag/flag01:/bin/bash

# the hashed password is: '42hDRfypTqqnw'

# in the host machine (where I have sudo access) I will install and use 'John the ripper' to crack the hashed password
host-machine$ john level01/resources/passwd.txt
